Setup Guide

five steps from git clone to a running substrate
built __BUILT_AT__ · this page is a guide; the CLI wizard does the work

Why a wizard at all?

The substrate's most powerful surfaces — Pan's panic gate, Zeus's ratification on Styx, Hephaestus's proposal pipeline — are by design CLI-and-Python only. The web UI would create attack surfaces those gates exist to prevent.

Setup is the operator's first ritual. The CLI wizard does it all in one terminal session; this page shows what's happening, in case you want to know first.

1Run the wizard

Idempotent — re-running is safe. Re-runs show current state.
$ invoke setup

2Kindle the hearth (Hestia)

Names this Olympus deployment. Every cognitive loop runs under this identity.
# Wizard prompts:
  name [my-olympus]: production-substrate
  vocation: private cognitive assistant for VANTA

3Choose your LLM provider

Olympus runs without an LLM. echo is a deterministic stub — safe by default. Switch to anthropic for real agent reasoning (Claude). The wizard tests the API call before saving — no surprise broken setups.
# Choices:
  1. echo       # no network, no cost (recommended to start)
  2. anthropic  # Claude (needs ANTHROPIC_API_KEY)
  3. skip       # leave as-is
  choice (1/2/3) [echo]: 2
  ANTHROPIC_API_KEY (paste): sk-ant-…
  testing the connection (one tiny call)…
  ✓ test call succeeded (model=claude-opus-4-7)

4Install the daemon (optional)

launchd on macOS or systemd on Linux runs invoke session + improve on a cadence. You can install now or skip and add later via invoke daemon install.
  install daemon now (Y/n): n
  # Will run interactively / via cron for now

5Pick the Agora port + first session

Agora binds to localhost:8765 by default. The wizard runs one session to confirm everything works.
  Agora port [8765]: [enter]
  run a session now (Y/n): [enter]
  ✓ session session-abc12345 completed
    hydra=9 argos=9 proposals=0

6Open the Agora

From now on you can use the web UI for everyday operator work. The CLI is always available for the constitution-bearing actions.
$ invoke serve --port 8765 &       # start the API
$ invoke agora --open               # build + open Agora

Where the config lives

state/config.json — gitignored, plaintext, operator-owned. The wizard writes it; env vars override it; the LLM bridge auto-loads it.

API keys are stored as plaintext. The operator is responsible for filesystem-level security. A future arc may add OS-keychain integration. We don't pretend otherwise.

Run invoke setup in your terminal to begin.
