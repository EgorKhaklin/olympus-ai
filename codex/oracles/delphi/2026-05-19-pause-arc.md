## What ships

### Fix #1 — `tests/conftest.py` adds a session-scoped real-state guard

The mechanism that would have caught my Hades mistake:

- At pytest session start: snapshot the sha256 of every file in `state/` matching `*.json`, plus the project's `state/config.json` specifically.
- At session end: re-snapshot. If ANY file's sha changed, fail the entire session with a clear error naming the offending file(s).
- The guard is opt-out via env var `OLYMPUS_TEST_ALLOW_STATE_WRITE=1` for cases like the Plutus seed-write integration test that legitimately writes one `llm.call` record.

The guard does NOT prevent the write — by the time pytest could intercept, the damage is done. It just **detects** + **fails loudly**. The discipline is "don't let a contaminated suite pass green."

### Fix #2 — `test_default_is_echo` cleans state properly

Patch `_path` directly via monkey-patch to point at `tmp_path / "config.json"` (the same pattern that fixed my Hades test). After this, the test passes regardless of the operator's real config.

### Fix #3 — `_check_session_errors` becomes a time-windowed metric

Today: counts errors among the last-50 / completed among the last-50. Math: 34 errors / 50 completed = 68%, stuck forever even as new clean sessions accumulate.

Replace with: count errors and completed that happened in the **last 24 hours** (or `OLYMPUS_DOCTOR_ERROR_WINDOW_SECONDS`). If denominator < 5 sessions, report "ok · insufficient data" instead of warning. The metric now reflects recent reality and clears itself naturally as the substrate runs cleanly.

### Fix #4 — surface deferred work as a single ARC-QUEUE.md doc

Rather than scattering TODOs through the code, the substrate now has one document at `codex/ARC-QUEUE.md` listing follow-up work named by the recent arcs. Initial population:

## What does NOT ship this arc

- **No automated `pyproject.toml` lint** for forbidden patterns (e.g. `monkeypatch.setenv("OLYMPUS_STATE_DIR")`). The guard catches the *consequence*; the pattern itself could be banned via a custom flake8 plugin or pre-commit hook — that's a follow-up arc.
- **No throne prompt-caching** — surfaced as ARC-QUEUE.md entry.
- **No retroactive test rewrites** for tests that *might* be sloppy — the guard catches them on next run; we don't preemptively rewrite hundreds of tests.
- **No new doctor checks beyond fixing `_check_session_errors`** — scope creep risk.
- **No re-deposit of the operator's API key** — operator must do that manually post-arc.
