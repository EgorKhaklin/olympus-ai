## What ships

### `invoke setup` — the interactive wizard (`runtime/setup.py`)

Idempotent. Re-running shows current state and asks "change?". Steps:

1. **Welcome** — a 5-sentence introduction to what Olympus is (the mythology as ontology, not decoration)
2. **Kindle the hearth** — deployment name + vocation; Hestia is lit
3. **LLM provider** — three honest choices:
   - **echo** (default, safe — no LLM calls, no network, all heuristics)
   - **anthropic** — Claude `claude-opus-4-7`; prompts for API key; **tests the call before saving** (no silent failures)
   - **skip** — leave unconfigured; can run later
4. **Daemon** — install the launchd/systemd unit? (yes/no/later)
5. **Web UI** — port for Agora (default 8765)
6. **First session** — runs `invoke session` once, shows the output, confirms loop works
7. **Summary** — what was configured, where the config lives, what to do next

Each step **explains what's happening and why**, then asks. Each step is recorded to `mnemosyne.remember(kind="setup.step")` so the operator can replay their setup choices.

Testable via injectable `input_provider` — production uses `input()`; tests pass canned answers.

### `state/config.json` (`runtime/config.py`)

Single JSON file. Fields:

```json
{
  "kindled": "my-olympus",
  "vocation": "production substrate",
  "llm": {
    "provider": "anthropic",
    "anthropic_api_key": "sk-ant-..."   // operator's choice; gitignored
  },
  "daemon": {"installed": true, "interval_seconds": 600},
  "agora": {"port": 8765},
  "setup_completed_at": "2026-05-19T...",
  "version": "0.1.0"
}
```

Lives in `state/` (already gitignored). The bridge reads from `OLYMPUS_LLM` env var first, then config; if neither set, defaults to echo.

**Operators who use env vars are unaffected.** The config file is additive.

### Agora — `src/olympus/agora/` (NEW package)

Vanilla HTML/JS web pages served as static files; consumed via the existing read-only HTTP API. Five pages:

- **`/agora/`** — operator dashboard (substrate status, harmony, today oracle)
- **`/agora/setup`** — visual setup guide (shows what `invoke setup` does, step-by-step)
- **`/agora/doctor`** — doctor diagnostic with explanations of each warn/fail
- **`/agora/today`** — single-action oracle as a focus card
- **`/agora/agents`** — agent invocation form (pick role, type prompt, see structured output)

Built via `invoke agora` (writes to `state/agora/` like Iris). Operator opens `state/agora/index.html` in browser; pages poll the HTTP API every 5s.

**The constitution holds.** Agora is presentation; the HTTP API is read-only (with the one POST /proposals/raise gate); the constitution-bearing endpoints (Pan, Momus, Delphi, Zeus) are CLI/Python-only by design.

### `codex/QUICKSTART.md` — 5-minute tour

Written for an outside observer. Covers:
- What Olympus is (one paragraph, no jargon)
- `git clone → pip install → invoke setup`
- The three LLM provider choices and which to pick
- How to see what the substrate is doing (Agora + doctor)
- Where to go for depth (the codex docs, in reading order)

### Bridge auto-loads config

`runtime/llm_bridge.py::bridge()` already honors `OLYMPUS_LLM`. New behavior: if env var unset, it reads `state/config.json::llm.provider` and `llm.anthropic_api_key` (the latter is set as the SDK's env var only for this process). **Env var still wins** — operators who set it explicitly are honored.

| language | role | why |
|---|---|---|
| Python (stdlib) | every module | discipline holds |

| JSON | `state/config.json` | standard, human-readable |

**No new language this arc.** The Iris dashboard already established the HTML/JS pattern; Agora extends it with interactive forms (still client-side only — actions either generate `invoke ...` commands the operator copies OR POST to existing API endpoints).

## What does NOT ship

- **No web-based ratification of HIGH-risk actions.** S7 holds. Zeus's authority comes via Styx oath signed by the operator in person. Agora *displays* pending proposals; the operator runs `invoke action ratify <id>` (the action queue's existing CLI surface).
- **No GUI installer.** The CLI wizard is the canonical install path. Agora's `/agora/setup` page is a *guide*, not a replacement.
- **No "config from URL" remote provisioning.** Config edits are local-only — the operator writes their own file (or `invoke setup` does it for them).
- **No API key encryption at rest** in this arc. `state/config.json` is gitignored; the operator is responsible for filesystem-level security. Future arc may add OS keychain integration. Documented explicitly so the operator isn't surprised.
