## What ships

### Hygieia — `src/olympus/olympians/hygieia.py`

In myth: daughter of Asclepius (the healer) and Epione; personification of **health, wellness, and the maintenance of well-being**. Where Asclepius cures the sick, Hygieia keeps the healthy *healthy*.

In Olympus, Hygieia is the **whole-substrate cohesion checker**. Asclepius rebuilds derived state. Hygieia asks the deeper question: *"are the modules' views of the world still consistent with each other?"* Specifically:

- **Pan ↔ recent invariants:** Does Pan's current panicked-state match the actual recent rate of `invariant.violated`?
- **Atlas ↔ session.completed:** Does Atlas's burden count match the number of unreleased sessions?
- **Daedalus ↔ disk:** Does every `_COGNITIVE_FLOW` node correspond to a real module?
- **Themis ↔ records:** Do recent Mnemosyne records still pass their JSON Schemas?
- **Plato ↔ figures:** Are there figures on disk that the taxonomy doesn't classify?
- **Charon ↔ Atlas:** Are there old released burdens that should have been ferried?

Each check returns a `HygieiaFinding`; the report aggregates them. Any non-`well` finding becomes a candidate Hephaestus proposal.

### Phoenix — `src/olympus/heroes/phoenix.py`

In myth: the firebird of cyclical death and rebirth. It lives long, burns itself in a great fire, and rises from the ashes — the same bird, renewed.

In Olympus, Phoenix is the **cyclical-regeneration primitive**. Distinct from:
- **Asclepius** (rebuilds *derived* state — Iris HTML, Pan state file, dirs)
- **Charon** (archives *completed* state — old Atlas burdens move to Hades)
- **Prometheus** (improves substrate per registered handlers)

Phoenix identifies state that has *completed a lifecycle* and is *due to be reborn*:

- A retired prophecy whose retirement reason no longer holds → ready for re-prediction
- A hung Atlas burden whose owner is gone → ready for release-and-restart
- A graduated prophecy that hasn't been verified recently → cycle re-engages
- A panic state that has been cleared but whose underlying invariants still fire occasionally → ready for a recalibrated threshold

Phoenix never directly acts. She surfaces "candidates for rebirth" as `phoenix.candidate` records; Hephaestus can then propose, Momus contest, Zeus ratify.

### Daedalus — graph centrality analysis

Daedalus's `_COGNITIVE_FLOW` is already a directed graph. The phi arc made the diagram beautiful; this arc makes the graph *queryable*. Two additions:

- `daedalus.centrality()` — computes betweenness centrality (the fraction of shortest paths through each node) over the cognitive-flow graph. Reveals which figures are most load-bearing in the actual flow, not by reputation.
- `daedalus.degree()` — in/out degree per node.

Operators can ask: *"what is the most load-bearing node in the substrate's reasoning?"* — and get a numerical answer.

### Euterpe — `src/olympus/muses/euterpe.py`

In myth: the Muse of music, lyric poetry, and song. Pythagoras (via the *monochord*) discovered that consonant musical intervals correspond to simple integer ratios: octave 2:1, perfect fifth 3:2, perfect fourth 4:3, major third 5:4, minor third 6:5.

In Olympus, Euterpe is **musical-interval consonance scoring** — a complement to Pythagoras's φ-harmony. Where Pythagoras measures distance from {φ, 1/φ, 1, 2}, Euterpe measures distance from the consonant musical intervals. A ratio of 1.5 (perfect fifth) gets a high Euterpe score; the same ratio gets a moderate Pythagoras-harmony score (close to 1 but not φ). The two scoring systems are complementary perspectives on the same substrate ratios.

### `invoke today` — the single-action oracle

`runtime/today.py` produces ONE concrete operator action from across the substrate's recent state:
- If Pan is in panic → "address the panic first"
- Else if Cassandra has a vindication you haven't acted on → "consider re-raising the dismissed warning on slice X"
- Else if Metis has a high-confidence recommendation → "consider tuning parameter Y to Z"
- Else if Phoenix has a rebirth candidate → "consider regenerating X"
- Else → "the substrate is healthy; quiet days are good days"

One action. Not a digest. The dashboard already shows the digest.

### Iris live mode

`invoke iris --live` writes a *self-refreshing* HTML page that polls `/status` every 5 seconds via `XMLHttpRequest` (no framework, no WebSocket — same vanilla-JS discipline as static Iris). Operators get a genuinely-live view. Static `index.html` remains as the snapshot view; `live.html` is added.
