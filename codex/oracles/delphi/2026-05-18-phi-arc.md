## Methodology — when is sacred geometry not decorative?

AP8 (decorative additions claiming structural value) applies. The discipline: every addition must answer *"what would change if it spoke?"* — exactly as it did for the missing-figures arc.

Three things in this arc are non-decorative:

1. **The golden-section search algorithm** — a real optimization technique that finds the extremum of a unimodal function in O(log n) calls. Metis currently advises parameter changes by heuristic ("if panic frequency ≥ 5, raise threshold from 3 → 5"). Golden-section search lets her *find* the right value within a range. Mathematical, classical, useful.

2. **The Fibonacci backoff curve** — well-known to scale better than exponential backoff for retries because the ratio between successive delays approaches φ (≈ 1.618) instead of 2.0, giving smoother load distribution. Hecate currently has fixed retry logic; Fibonacci backoff is a real improvement.

3. **Plato's five-solid taxonomy** — provides a *second navigational axis* through the pantheon. Today figures are organized by *tier* (Olympian, Titan, Hero, …). Plato organizes them by *function* (observation / state / reasoning / authority / execution), each mapped to one of the Platonic solids. This isn't redundant — tier asks "what kind of figure," function asks "what does it do." Both useful for different queries.

Three things in this arc are admittedly aesthetic but earn their place by being *legibility wins*:

4. **Metatron's Cube SVG diagram** — 13 nodes for the canonical 13 (Olympians + Hestia), edges showing the cognitive-flow relationships from Daedalus. Renders inline in GitHub markdown. Operators can *see* the canonical pantheon at a glance.
5. **Vesica Piscis SVG diagram** — 2 intersecting circles showing where two domains overlap. Used to render "where does Athena's brief intersect Hephaestus's proposal."
6. **Harmony scoring** — given a ratio (e.g., ratified/proposed), Pythagoras reports its proximity to φ, 1/φ, or 1. Surfaces a numerical aesthetic — *"the substrate is in harmony when the acceptance rate sits near 1/φ."*

#6 is the most aesthetically-motivated. Its load-bearing-ness: it gives a single-number summary the operator can track over time. AP8 contests would fire if it claimed to *prove* anything; it doesn't — it scores.

## Q1 — What ships

### Pythagoras — `src/olympus/heroes/pythagoras.py`

In history: 6th-century-BCE mathematician whose school treated number as the foundation of reality. He formalized the proportional relationships that became the golden ratio (then called "the section in extreme and mean ratio").

In Olympus, Pythagoras is the **sacred-numerics module**. He exposes:

- **Constants:** `PHI`, `PHI_INVERSE`, `PI`, `E`, `SQRT2`, `SQRT3`, `SQRT5` — canonical mathematical constants for any module needing them
- **Sequences:** `fibonacci(n)` returns the nth Fibonacci number; `fib_sequence(k)` returns the first k
- **Backoff:** `fib_backoff(attempt, *, base_seconds, cap_seconds)` returns the Fibonacci-scaled retry delay for a given attempt
- **Search:** `golden_section_search(fn, lo, hi, *, tol)` finds the minimum of a unimodal function in O(log((hi-lo)/tol)) calls
- **Harmony:** `harmony(ratio)` returns a 0..1 score for proximity to φ, 1/φ, or 1 — the "harmonic distance"
- **Triples:** `pythagorean_triples(below)` generates all (a,b,c) where a²+b²=c²; useful for spatial validation

He records every `golden_section_search` call to Mnemosyne under `pythagoras.search` so the optimization history is reconstructable (S8).

### Plato — `src/olympus/heroes/plato.py`

In history: philosopher who described the five regular convex polyhedra in *Timaeus*; each was associated with a classical element. The mapping has held in mathematics ever since.

In Olympus, Plato is the **five-solid taxonomy of substrate work**:

| solid | vertices | element | function | examples |
|---|---:|---|---|---|
| **Tetrahedron** | 4 | fire | observation | Hydra, Argos, Furies, Pythia |
| **Cube** | 8 | earth | state | Mnemosyne, Styx, Atlas, Charon, Hades, Hera, Iapetus |
| **Octahedron** | 6 | air | reasoning | Athena, Hephaestus, Momus, Epimetheus, Cassandra, Nemesis |
| **Dodecahedron** | 12 | cosmos | authority | Zeus, Themis, Pan, Asclepius, Metis, Daedalus, Hecate |
| **Icosahedron** | 20 | water | execution | Prometheus, ActionQueue, Castor, Pollux, Ariadne, Daemon, HTTP-API, Federation, Iris |

`plato.classify(name)` returns which solid a figure belongs to. `plato.cosmos()` returns the full mapping. Each figure's classification is hand-maintained, like Daedalus's `_COGNITIVE_FLOW` — drift between the taxonomy and the actual modules is a Hephaestus signal.

### Daedalus — sacred-geometry SVG diagrams

Daedalus already emits Mermaid diagrams of the cognitive flow. The phi arc extends him with **two inline SVG diagrams** embedded directly in `codex/ARCHITECTURE.md`:

- **Metatron's Cube** — 13 circles for the canonical 13 Olympian figures (12 Olympians + Hestia), connected by every-vertex-to-every-vertex edges; centered on the cosmic axis
- **Vesica Piscis** — 2 intersecting circles labeled with the two domains whose overlap is being illustrated (e.g., Athena ∩ Hephaestus, Pythia ∩ Mnemosyne)

SVG is XML and renders natively on GitHub when embedded in markdown. **No build step, no new toolchain** — the SVG is text in the markdown file.

### Metis uses golden-section search

Metis currently produces parameter recommendations by heuristic rules. The phi arc adds a new advisor pathway: for any *numerical* parameter (Pan threshold, daemon interval, Charon retention), Metis can call `pythagoras.golden_section_search` with an evaluation function to *find* a recommended value, not just guess one. The evaluation function aggregates recent outcomes — fewer panics + fewer hung-burdens + lower iteration duration → higher score.

### Hecate uses Fibonacci backoff

Hecate's retry path currently uses fixed timing. The phi arc replaces this with `pythagoras.fib_backoff(attempt)` for the standard exponential-backoff problem with a smoother growth curve. This is a *real engineering improvement*, not a decorative one.

### HTTP `/geometry` route + `invoke harmony` + `invoke geometry`

The HTTP API gains `GET /geometry` returning:
- Plato's taxonomy (figure → solid → element → function)
- Pythagoras's recent harmony scores
- The Metatron's Cube node list

CLI surfaces: `invoke geometry` shows Plato's taxonomy; `invoke harmony` shows current substrate ratios with their harmony scores; `invoke pythagoras --search <fn>` exercises the golden-section search.

## Q2 — Languages used

| language | role | why |
|---|---|---|
| **SVG** (XML inside markdown) | Metatron's Cube + Vesica Piscis | renders natively on GitHub; text-based source of truth; no toolchain |
| Python (stdlib only) | every module | same discipline |

**SVG-in-markdown is the new format this arc.** It's not really a new language — it's XML, which we've already used for the launchd plist — but applied in a new context. Pure stdlib emits it; GitHub renders it. The right tool.

Refused:
- **LaTeX** for mathematical typesetting — GitHub doesn't render LaTeX without special setup; markdown's math support via $$...$$ is partial. SVG renders cleaner.
- **sympy / numpy** — every Pythagoras function is implementable in stdlib. Adding a numerical dep would be AP8.
